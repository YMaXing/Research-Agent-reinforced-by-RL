    are exact or over-predict; only 8% under-predict).
  - Regression-to-middle pattern: low-oracle sections get inflated toward
    P2-P3; high-oracle sections (P4, P5) get deflated toward P3-P4.
  - Over-shoots are cheap: higher presets cluster tightly in reward space,
    so a +1/+2 overshoot typically costs < 0.05 reward.
  - Under-shoots on P4/P5 are expensive (loss 0.08-0.56); the reward
    function penalises complexity under-estimation more severely.
  - One catastrophic miss: 06_tools__var_demanding section
    "Implementing a tool calling framework from scratch"  (P5→P1, loss=0.56)
    accounts for a disproportionate share of total E[R] gap.

========================================================================
 K. Headline conclusions for retraining strategy
========================================================================
  1. The model picks top-2 by reward in 81% of sections — it has the
     right neighbourhood, wrong cell.  Improvement should target
     near-ties, not raw classification.
  2. Class imbalance (P0=33% vs P4=7.5%) drives regression-to-middle.
     Variant-stratified oversampling or per-class loss weighting needed.
  3. Half the sections are within 0.03 reward — the reward function
     itself is the bottleneck.  Tightening it (extra signal, sharper
     margins) is more leveraged than scaling the model.
  4. Variant signal is unreliably attended-to (see 06_tools disaster).
     A variant-aware curriculum (or stronger conditioning prompt) would
     help the model differentiate same-content / different-variant cases.
  5. The single P5→P1 catastrophe accounts for ~20% of the E[R] gap.
     Worth investigating whether the variant token is reaching the
     prompt correctly for this specific article.
